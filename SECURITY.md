# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch and the active `release` branch.

## Reporting a Vulnerability

1. Do not open a public GitHub issue.
2. Email vulnerability details to the maintainers with reproduction steps and impact.
3. Include affected commit/version and suggested remediation if available.

The maintainers will acknowledge reports within 3 business days and provide a remediation status update within 10 business days.

## Disclosure Process

- Critical/high findings are patched first and released as soon as validation passes.
- Public disclosure happens after a fix is released or a coordinated disclosure date is agreed.
